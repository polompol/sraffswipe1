# StaffSwipe — память проекта для Claude

Telegram Mini App «Tinder для подработок в общепите» (Москва). Двусторонний
маркетплейс: соискатели (официант/бариста/повар...) ↔ заведения. Владелец
проекта — не разработчик: объясняй просто, по-русски, без жаргона.

## Структура

- `tma/` — клиент: React 18 + TS + Vite + @telegram-apps/sdk-react v3,
  TanStack Query, Zustand, HashRouter. Без backend работает на mock
  (`src/api/mock.ts` зеркалит `src/api/endpoints.ts`).
- `backend/` — FastAPI + SQLAlchemy 2 + Pydantic v2, SQLite (dev/тесты) /
  PostgreSQL (прод), Alembic. `backend/bot/` — aiogram-бот (отдельный процесс).
- `docs/` — LAUNCH.md (запуск), DEPLOY.md, COMMISSION.md (деньги/споры),
  OPERATIONS.md (операционка), legal/ (оферта, 152-ФЗ).
- Деплой: `docker-compose.prod.yml` (Caddy HTTPS + Postgres + API + бот + SPA).

## Команды проверки (запускать после ЛЮБЫХ правок)

```bash
# Backend (из backend/)

npm run lint && npx tsc --noEmit && npx vitest run && npm run build
# Всё сразу: bash scripts/verify.sh
```

Новые поля моделей = новая миграция Alembic (`backend/migrations/versions/`,
ревизии — hex-цепочка, head один).

## Ключевая доменная логика (не ломать!)

- **Смена закрывается ТОЛЬКО взаимным подтверждением**: работник («я на
  смене»: 6-значный код `secrets` ИЛИ гео ≤0.3км) + заведение («человек
  пришёл»). Конфликт → спор → оператор решает (completed|no_show).
- **Комиссия** (10%, `_accrue_commission`): только за закрытую смену,
  идемпотентно по match_id. Сначала автосписание с баланса (атомарный
  UPDATE с условием), иначе pending → недельный счёт. Просрочка >7 дней →
  блок публикации вакансий И позитивных свайпов заведения.
- **Баланс** (`Entitlement.balance_rub`) — учёт аванса; каждое движение в
  `wallet_txns`. Пополнение: оператор в админке или ЮKassa-вебхук
  (идемпотентен по charge_id, сумма сверяется с metadata).
- **Идентичность = tg_id**. Потеря аккаунта → `/admin/relink` (перенос на
  новый tg_id; на tg_id админа — запрещено, это эскалация прав).
- Деньги/балансы списывать только атомарными UPDATE с условием — никаких
  read-modify-write.

## Правила проекта

- Комментарии в коде и тексты UI — по-русски. Тон UI: простой, тёплый.
- Дизайн: багровый `--gold:#a51c30` + слоновая кость `--bg:#efe7d3`, токены
  в `tma/src/theme/theme.css`, тёмная тема `[data-theme="dark"]` — новые
  цвета только через токены.
- НЕ хранить фото документов (медкнижки и т.п.) — только статусы. 152-ФЗ.
- ИНН/телефон не отдавать в публичных эндпоинтах (лента кандидатов — без них).
- Секреты только через env; `assert_production_safe()` не ослаблять.
- Playwright и скриншоты — временно, в репозиторий не коммитить.
- Не продавать нереализованные фичи (SKU без эффекта не включать в каталог).
- Каждая новая фича перед коммитом — краткий взгляд злоумышленника: что тут
  можно накрутить/обойти/подобрать?

## Состояние

Продукт готов к пилоту, код в `main`. Ожидает: сервер + домен + токен бота
(см. docs/LAUNCH.md). Автосписание ЮKassa доступно самозанятому БЕЗ ИП (чек —
в «Мой налог», yookassa_send_receipt=false). Отложено до триггеров:
эскроу, селфи-чекин, авто-закрытие смен по таймеру.
