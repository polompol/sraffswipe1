# StaffSwipe — Tinder для подработок в общепите 🇷🇺

Сервис, где соискатели (официант, бариста, повар, посудомой, хостес, бармен) и
работодатели (кафе, бар, ресторан, кофейня) свайпают карточки. При взаимном
лайке — **мэтч → чат → подтверждение смены → PDF-акт** для самозанятого.

## Из чего состоит проект

| Компонент | Стек |
| --- | --- |
| **`tma/`** — Telegram Mini App (клиент) | React + TS + Vite + @telegram-apps/sdk-react |
| **`backend/`** — API + бот | FastAPI + SQLAlchemy + aiogram 3 |

Вход через Telegram (без SMS), нативные кнопки/хаптика, оплата внутри Telegram.

```bash
# Backend
cd backend && pip install -r requirements.txt && uvicorn app.main:app --reload

# Telegram Mini App
cd tma && npm install && npm run dev
```

Проверка всего проекта одной командой: `bash scripts/verify.sh`.

## Что реализовано

- Онбординг, выбор роли, профиль (роли, медкнижка, самозанятость, фото).
- **Свайп-лента** и альтернативный список; фильтры (роль, ставка, дата, без
  медкнижки/опыта, только проверенные, сортировка).
- Мэтч → **чат** (REST + WebSocket) → подтверждение смены обеими сторонами.
- **PDF-акт** выполненных работ для самозанятого.
- Монетизация: подписки (ЮKassa) + микро-фичи (Telegram Stars), боост вакансий,
  супер-лайки, premium, верификация заведения (DaData).
- Рост: рефералы, сохранённые поиски с алертами бота, стрик, шеринг.
- Тёмная тема, аналитика воронки, согласие 18+ / 152-ФЗ.

### Монетизация (гибридная модель)

| Источник | Что | Платёж |
| --- | --- | --- |
| Подписки работодателей | Free / Pro (1 990 ₽/мес) / Business (4 990 ₽/мес) | ЮKassa |
| Boost вакансии | поднятие в топ ленты | Telegram Stars |
| Супер-лайки «Срочно» | пакеты 5/20 | Telegram Stars |
| Premium соискателя | «кто меня лайкнул», приоритет | Telegram Stars |
| Верификация заведения | бейдж «Проверен» (DaData) | ЮKassa |

Цифровые микро-фичи внутри Telegram — через **Stars** (требование Telegram),
B2B-подписки/верификация — через **ЮKassa** (рубли). Backend ведёт права
(`entitlements`), начисление идемпотентно через `/billing/fulfill`.

## ⚠️ Прод-конфиг: вставить ключи и запустить

Все значения берутся из env (см. **`backend/.env.example`** и **`tma/.env.example`** —
скопируйте в `.env` и заполните). В прод-режиме (`DEV_MODE=false`) приложение
**не стартует** с дефолтными секретами — это защита от случайного небезопасного
деплоя.

| Переменная | Зачем |
| --- | --- |
| `JWT_SECRET` | подпись токенов (обязательно сменить) |
| `INTERNAL_API_SECRET` | защита `/billing/fulfill` и вебхука (бот/ЮKassa) |
| `ALLOW_INSECURE_TELEGRAM_AUTH=false` | запретить вход без подписи initData |
| `TELEGRAM_BOT_TOKEN` | валидация initData, инвойсы Stars, уведомления |
| `ALLOWED_ORIGINS` | CORS: URL Mini App вместо `*` |
| `YOOKASSA_SHOP_ID` / `YOOKASSA_SECRET_KEY` | приём рублёвых платежей |
| `DADATA_TOKEN` | верификация ИНН/ОГРН и подсказки адресов |
| `S3_*` | хранилище фото (Yandex Object Storage / S3) |

Безопасность (в коде): доступ к чату/акту/подтверждению только участникам мэтча;
лента кандидатов — только работодателю, без ИНН; акт — только после подтверждения
смены; идемпотентные свайпы и уникальные ключи в БД (нет двойных списаний/мэтчей);
рейт-лимит на запись и на SMS-коды (срок жизни кода 10 мин); строгая валидация
входных данных; неблокирующие уведомления.
