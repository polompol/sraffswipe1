# StaffSwipe — полная инструкция запуска (по шагам, с сайтами)

Это подробный путь от «ничего нет» до «приложение живое в Telegram».
Время: ~1–2 часа активных действий. Деньги: ~500–1000 ₽ (сервер + домен).

У тебя уже есть: **токен бота** и **домен**. Если чего-то нет — шаги 1–2
объясняют, где взять.

---

## Шаг 0. Что понадобится под рукой

- Телефон с Telegram (твой аккаунт).
- Банковская карта (оплатить сервер/домен).
- Ноутбук/ПК удобнее, но можно и с iPhone: поставь приложение
  **Termius** (App Store, бесплатно) — это SSH-клиент, через него
  вводят команды на сервере.

---

## Шаг 1. Бот в Telegram (если ещё не создан)

1. Открой Telegram → поиск → **@BotFather** (галочка верификации).
2. Отправь команду: `/newbot`
3. Он спросит имя — введи: `StaffSwipe`
4. Он спросит username — введи что-то свободное, оканчивается на `bot`,
   например: `staffswipe_bot`
5. BotFather пришлёт **токен** вида `1234567890:AAE...xyz` —
   **скопируй и сохрани**, никому не показывай.

## Шаг 2. Сервер (VPS)

Подойдёт любой из этих сайтов (все российские, платёж картой РФ):

- **timeweb.cloud** ← рекомендую, просто и дёшево
- selectel.ru
- beget.com

По шагам на Timeweb:
1. Зайди на **timeweb.cloud** → «Регистрация» (почта + пароль).
2. В панели: **«Облачные серверы» → «Создать сервер»**.
3. Образ ОС: **Ubuntu 22.04**.
4. Конфигурация: **2 CPU / 2 ГБ RAM / 30 ГБ диск** (~400–600 ₽/мес).
5. Регион: Москва (или Санкт-Петербург).
6. Создай сервер → через 1–2 минуты в панели появятся:
   **IP-адрес** (вида `185.xx.xx.xx`), логин **root** и **пароль**.
   Сохрани все три.

## Шаг 3. Домен → на сервер

Домен у тебя есть. Зайди в панель регистратора (где покупал домен —
reg.ru / timeweb / nic.ru), найди раздел **«DNS» / «Управление зоной»**:

1. Создай запись типа **A**:
   - Имя (хост): `app` (получится `app.твойдомен.ру`)
   - Значение: **IP сервера** из шага 2
   - TTL: оставь по умолчанию
2. Сохрани. Подожди 5–15 минут.
3. Проверка: на телефоне/ПК открой терминал (или сайт **dnschecker.org**,
   введи `app.твойдомен.ру`) — должен показать IP твоего сервера.

## Шаг 4. Юридические документы (152-ФЗ) — до публичного запуска

В репозитории лежат заготовки: `docs/legal/` (оферта и политика ПДн).
Опубликовать проще всего на **telegra.ph**:

1. Открой **telegra.ph** в браузере.
2. Заголовок: «Политика обработки персональных данных StaffSwipe» →
   вставь текст из `docs/legal/privacy.md` → нажми **Publish**.
   Скопируй получившуюся ссылку.
3. То же самое для оферты (`docs/legal/offer.md`) → вторая ссылка.
4. Ссылки понадобятся на шаге 6 (переменные `VITE_PRIVACY_URL` и
   `VITE_OFFER_URL`).

⚠️ Перед этим вычитай документы и, как ты и планировал, покажи юристу.
Для закрытого теста с знакомыми можно запустить и без ссылок, но для
публичного запуска они обязательны.

## Шаг 5. Подключиться к серверу и поставить Docker

С ПК (Windows: PowerShell; macOS: Терминал) или с iPhone через Termius:

```sh
ssh root@185.xx.xx.xx        # вместо xx — IP из шага 2, пароль пришлёт хостинг
```

Дальше по одной команде (копируй построчно):

```sh
curl -fsSL https://get.docker.com | sh
```
(ставится 1–2 минуты)

```sh
ufw allow 80 && ufw allow 443 && ufw allow 22
```
(открыть веб-порты; если ответит «command not found» — пропусти, фаервол выключен)

## Шаг 6. Скачать код и заполнить настройки

```sh
git clone https://github.com/polompol/sraffswipe1.git
cd sraffswipe1
```

Сгенерировать секреты (выведет три случайные строки):

```sh
bash scripts/gen-secrets.sh
```

Создать файл настроек и открыть редактор:

```sh
cp .env.example .env
nano .env
```

В редакторе заполни (стрелками ходишь, печатаешь как обычно):

| Строка | Что вписать |
|---|---|
| `DOMAIN=` | `app.твойдомен.ру` (без https://) |
| `TELEGRAM_BOT_TOKEN=` | токен из шага 1 |
| `BOT_USERNAME=` | username бота без @ (напр. `staffswipe_bot`) |
| `POSTGRES_PASSWORD=` | 1-я строка из gen-secrets.sh |
| `JWT_SECRET=` | 2-я строка |
| `INTERNAL_API_SECRET=` | 3-я строка |
| `ADMIN_TG_IDS=` | уже вписан твой id `7833280774` — не трогай |
| `VITE_OFFER_URL=` | ссылка на оферту с telegra.ph (шаг 4) |
| `VITE_PRIVACY_URL=` | ссылка на политику с telegra.ph |

**Твой Telegram-id** уже вписан в шаблон (`7833280774`) — админ-панель
будет доступна только тебе. Захочешь добавить второго оператора — допиши
его id через запятую (id узнаётся у бота **@userinfobot**).

Сохранить и выйти из nano: **Ctrl+O → Enter → Ctrl+X**.

## Шаг 7. Запуск (одна команда)

```sh
docker compose -f docker-compose.prod.yml up -d --build
```

Первый запуск — 3–5 минут (собираются образы, берётся HTTPS-сертификат).

Проверка, что всё поднялось:

```sh
docker compose -f docker-compose.prod.yml ps
```
— у всех сервисов статус Up / Healthy.

```sh
curl https://app.твойдомен.ру/api/health
```
— должен ответить `{"status":"ok"}`.

Если что-то не так — смотри логи:
```sh
docker compose -f docker-compose.prod.yml logs -f api
docker compose -f docker-compose.prod.yml logs -f caddy
```

## Шаг 8. Подключить Mini App к боту

1. Telegram → **@BotFather** → `/mybots` → выбери своего бота.
2. **Bot Settings → Menu Button → Configure menu button**.
3. Пришли URL: `https://app.твойдомен.ру`
4. Пришли текст кнопки: `Открыть`

Проверка: напиши своему боту `/start` → появится кнопка →
нажми → **приложение откроется внутри Telegram**. 🎉

## Шаг 9. Смоук-тест (5 минут)

- [ ] Онбординг открывается, регистрация как **соискатель** проходит.
- [ ] Профиль открывается; внизу видна кнопка **«Админ-панель»**
      (значит `ADMIN_TG_IDS` вписан верно).
- [ ] Попроси знакомого (или второй аккаунт TG) зайти как **заведение**
      и создать тестовую смену.
- [ ] Соискателем свайпни её вправо → мэтч → чат работает.
- [ ] Обе стороны подтвердили → в день смены работает «Я на смене» /
      «Человек пришёл».

Всё прошло — **ты в проде**.


## Частые проблемы

| Симптом | Причина и решение |
|---|---|
| Сертификат не выдаётся / сайт не открывается | A-запись не указывает на сервер или порты 80/443 закрыты. Проверь dnschecker.org и `ufw status`. Логи: `logs -f caddy` |
| `/api/health` отвечает 502 | API не поднялся — чаще всего не заполнены секреты в `.env`. Логи: `logs -f api` |
| Кнопка в боте не открывает приложение | В BotFather URL должен быть ровно `https://app.твойдомен.ру` — с https, без слэша в конце |
| Нет кнопки «Админ-панель» | В `ADMIN_TG_IDS` не тот id — сверься с @userinfobot, перезапусти: `docker compose -f docker-compose.prod.yml up -d` |
| Поменял `.env` | Изменения применяются после `docker compose -f docker-compose.prod.yml up -d --build` |

## Что подключается ПОЗЖЕ (не для пилота)

- **Приём денег (ЮKassa)** — когда оформишь ИП/ООО: зарегистрируйся на
  **yookassa.ru**, получи `SHOP_ID` и `SECRET_KEY`, впиши в `.env`
  (`YOOKASSA_SHOP_ID`, `YOOKASSA_SECRET_KEY`, отдельный
  `YOOKASSA_WEBHOOK_SECRET`), перезапусти. До этого оплата смен идёт
  напрямую заведение ↔ работник, твоя комиссия — по договорённости.
- **Хранилище фото (S3)** — Yandex Object Storage (cloud.yandex.ru):
  ключи в `S3_*`. Без них всё работает, кроме загрузки фото.

## Первая неделя после запуска (главное!)

1. Зайди в 10 знакомых заведений своего района, покажи приложение,
   предложи: «не вышел человек — пришлю проверенную замену, 300–500 ₽
   за закрытие».
2. Собери 20–30 работников: чаты подработки в TG, штат этих же
   заведений, сарафан.
3. Считай одно число: **сколько смен реально закрыто за неделю**.
4. Споры по сменам приходят тебе в бота и в админ-панель — разбирай
   в тот же день, это твоя репутация.
