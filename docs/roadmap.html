StaffSwipe — план развития

Что делать дальше, что в коде оставить/убрать и как защититься. Обновлено: июнь 2026.

Где мы сейчас

Зрелый MVP в виде Telegram Mini App (React) + FastAPI backend + бот. Реализовано: вход через Telegram, свайпы, мэтчи, чат (WebSocket), смены, PDF-акт, монетизация (Telegram Stars + ЮKassa с вебхуком), рефералы, отзывы/рейтинг, бот-уведомления, аналитика воронки, расширенные фильтры, сохранённые поиски + алерты, список-вид, шеринг, тёмная тема, стрик, согласие 18+/152-ФЗ, рейт-лимит. Backend 32 теста + ruff, TMA eslint+tsc+vitest+build — зелёные, CI на 3 компонента.

не сделано — реальный запуск: нужны ключи (BotFather, ЮKassa, DaData, S3), PostgreSQL, хостинг РФ, пилот.

Дорожная карта

Этап 0 — Запуск пилота 0–2 недели

  • Поднять backend+бот на РФ-хостинге (Yandex Cloud / VK Cloud / Selectel), HTTPS.
  • PostgreSQL/PostGIS вместо SQLite; бэкапы.
  • Прописать ключи: TELEGRAM_BOT_TOKEN, YOOKASSA_*, DADATA_TOKEN, S3_*, JWT_SECRET, INTERNAL_API_SECRET; ALLOW_INSECURE_TELEGRAM_AUTH=false.
  • BotFather → Web App URL; вебхук ЮKassa.
  • Пилот: 1 город, 50–100 заведений, 1–2 месяца — проверить экономику и удержание.

Этап 1 — Удержание и контент 1–2 мес

  • Пуш-напоминания о смене за N часов (инфраструктура уведомлений уже есть).
  • Карта смен (Yandex Maps JS) — нужен ключ; координаты уже даёт DaData.
  • Реальные фото (S3 уже подключён) вместо заглушек; blurhash-плейсхолдеры.
  • Дашборд аналитики для админа (уже за ADMIN_TG_IDS) — графики когорт/retention.
  • Онбординг-видео, push-дайджест «смены недели».

Этап 2 — Доверие и масштаб фич 3–4 мес

  • No-show / споры: отметка «не пришёл», влияние на рейтинг, арбитраж.
  • Командные аккаунты работодателя (несколько HR на заведение/сеть).
  • Лидерборд топ-исполнителей, бейджи, расширенная геймификация.
  • Учёт графика доступности соискателя в ранжировании.
  • A/B-тесты экранов, реактивация «спящих» пушами.

Этап 3 — Платформа 6+ мес

  • Инфраструктура масштаба: Redis (рейт-лимит/кэш/чат pub-sub), очереди задач, PostGIS-индексы.
  • Мультигород/мультирегион, биллинг юрлиц, закрывающие документы.
  • Открытый API/вебхуки для агрегаторов вакансий и HR-систем.
  • Нативные сборки (RuStore/AppStore) из существующего Flutter-клиента.

Технический долг и улучшения кода

Качество

  • Больше тестов TMA (render-тесты ключевых экранов, не только логика).
  • Единый слой ошибок: ErrorBoundary (есть) + Sentry (хук есть) — включить DSN.
  • Вынести дубли (хелперы карточек), общий tool/_mockkit.py.
  • i18n (ru/en) — строки в словарь.

Производительность

  • Code-splitting уже есть; добавить кэш изображений + blurhash.
  • PostGIS-индексы для гео-радиуса вместо python-фильтра.
  • Пагинация ленты вместо полной выборки.
  • CDN для статики TMA и фото.

Безопасность — чеклист защиты

Уже в коде сделано

  • Валидация Telegram initData (HMAC по bot-token).
  • Свои JWT; /billing/fulfill только по X-Internal-Token.
  • Проверка участия в мэтче: PDF-акт, WebSocket-чат, подтверждение смены.
  • Серверный 18+, согласие 152-ФЗ/оферта.
  • Рейт-лимит на свайпы/сообщения/вакансии; CORS из списка.
  • Идемпотентность платежей (charge_id), супер-лайки/boost списываются.

Перед запуском обязательно

  • Сменить дефолтные JWT_SECRET / INTERNAL_API_SECRET.
  • ALLOW_INSECURE_TELEGRAM_AUTH=false; ALLOWED_ORIGINS = домен Mini App.
  • HTTPS везде; вебхук ЮKassa — проверка подписи/IP-allowlist.
  • PostgreSQL + ежедневные бэкапы + шифрование диска.
  • Cloudflare/WAF + анти-DDoS перед API.
  • Секрет-сканер и Dependabot в репозитории.

На масштабе

  • Пентест перед ростом (платежи + ПДн — высокий риск).
  • Анти-фрод: лимиты на создание вакансий/аккаунтов, скоринг спама, модерация контента.
  • RBAC и аудит-лог доступа к ПДн; экспорт/удаление данных по 152-ФЗ.
  • Rate-limit и сессии — в Redis; ротация ключей; секреты в Vault/Secret Manager.
  • Наблюдаемость: Sentry (ошибки) + метрики (Prometheus) + алерты.

Метрики, за которыми следить

Активация

open → первый свайп → отклик. Цель: ≥60% доходят до отклика.

Мэтч-рейт

отклик → мэтч → подтверждённая смена. Главный показатель пользы.

Удержание

D1/D7/D30, стрик, возвраты по пуш-алертам сохранённых поисков.

Деньги

конверсия в Pro/Business, ARPPU, доля boost/Stars, churn.

Виральность

K-фактор: приглашения (рефералы) + шеринг смен.

Доверие

доля проверенных заведений, средний рейтинг, no-show.

Риски

  • Холодный старт двустороннего рынка — сначала набрать заведения (платят), потом соискателей. Решение: ручной онбординг заведений в пилот-городе.
  • Экономика Stars↔рубли — комиссии Telegram и вывод; крупные платежи держать на ЮKassa.
  • Юридика РФ — самозанятость/НПД, ПДн: проверить документы у юриста до запуска.
  • Безопасность платежей/ПДн — без пентеста не масштабировать.
