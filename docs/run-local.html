Запустить НАСТОЯЩЕЕ приложение у себя за ~5 минут
Без сервера и ключей — на демо-данных. Чтобы потыкать живую версию.
Что это даёт: ты откроешь настоящее приложение (не прототип) в браузере на своём компьютере. Оно работает на демо-данных — регистрация, свайпы, мэтчи, чат, создание вакансии. Идеально, чтобы прокликать перед реальным запуском.

Что нужно один раз установить

1
Установи Node.js (версии 18+): зайди на nodejs.org → скачай кнопку LTS → установи как обычную программу (Далее-Далее-Готово).
2
Скачай код проекта: на странице репозитория кнопка Code → Download ZIP → распакуй. (Или, если умеешь, git clone.)

Запуск (3 команды)

3
Открой терминал:
  • Windows: в папке с проектом зажми Shift + правый клик → «Открыть окно PowerShell здесь».
  • Mac: приложение «Терминал», затем напиши cd и перетащи в окно папку tma, нажми Enter.
4
Перейди в папку приложения и установи зависимости (один раз): 
cd tma
npm install
Подожди 1–2 минуты — скачаются библиотеки.
5
Запусти: 
npm run dev
В окне появится строка вида Local: http://localhost:5173/.
6
Открой этот адрес http://localhost:5173/ в браузере (Chrome). Готово — это живое приложение. Тыкай: выбери роль, листай смены, лови мэтч, открой чат, создай вакансию.
Открыть на телефоне (по желанию): запусти npm run dev -- --host, в выводе появится адрес Network: http://192.168.x.x:5173/ — открой его в браузере телефона, который в той же Wi-Fi сети.

Как остановить / запустить снова

  • Остановить: в терминале нажми Ctrl + C.
  • Запустить снова: снова npm run dev (шаг 5). Команду npm install повторять не нужно.

Если что-то не так

  • Самое простое: в корне проекта лежит start-app.bat (Windows, двойной клик) и start-app.sh (Mac/Linux) — они сами всё установят и запустят.
  • «npm не найден» → Node.js не установился, повтори шаг 1 и перезапусти терминал (и VS Code, если запускаешь из него).
  • «выполнение сценариев отключено в этой системе» (PowerShell) → запусти через start-app.bat ИЛИ выполни один раз Set-ExecutionPolicy -Scope CurrentUser RemoteSigned.
  • «no such file or directory, open ... package.json» → команда запущена не из той папки: сначала cd tma, потом npm run dev.
  • Ошибки про доступ/EACCES (Mac) → попробуй sudo npm install.
  • Порт занят → Vite сам предложит другой адрес (например :5174), просто открой его.
  • Хочешь подключить к реальному серверу → в файле tma/.env укажи VITE_USE_BACKEND=true и адрес backend (но для «потыкать» это не нужно — демо-данные уже работают).
Это локальный запуск «для себя» — он виден только на твоём компьютере. Чтобы приложением пользовались люди в Telegram, нужен полноценный деплой по чек-листу launch.html (сервер, домен, ключи).
Соседние файлы: prototype.html (кликабельный макет), launch.html (запуск в прод), concept.html (концепт).
