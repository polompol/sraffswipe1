⚡StaffSwipe
Чек-лист запуска «по кнопкам» — от нуля до первых пользователей
Обновлено под мультигород · код готов, тесты зелёные
Главная мысль: код готов на ~90%. Дальше решает не программирование, а запуск и наполнение ленты руками. Этот файл — пошаговый план: что нажать, что куда вставить, как платить налоги, как продвигать и сколько можно заработать. Технические шаги (сервер) можно отдать любому программисту на 1–2 часа — команды уже готовы ниже.

📦 Что уже умеет проект

  • Вход через Telegram, выбор роли (соискатель / заведение), профиль 18+.
  • Свайп-лента и список смен, фильтры, фильтр по городу (новое).
  • Мэтч → чат → подтверждение смены обеими сторонами → PDF-акт для самозанятого.
  • Оплаты: подписки (ЮKassa) + супер-лайки/boost/премиум (Telegram Stars).
  • Рост: рефералы, сохранённые поиски с уведомлениями, стрик, шеринг.
  • Безопасность: доступы только участникам, валидация, анти-спам, защита платежей.
Мультигород решён: если рекламу увидят люди из разных городов — каждый видит ленту своего города (по профилю + переключатель «сменить город»). Пустую чужую ленту никто не получит.

🧭 Часть 0. Что подготовить заранее

  • Телефон с Telegram (для бота и тестов).
  • Компьютер ИЛИ знакомый программист на 1–2 часа (для шага «Сервер»).
  • Банковская карта / доступ в Госуслуги (для самозанятости и ЮKassa).
  • Название и логотип (можно простой — «StaffSwipe»).

🤖 Часть 1. Создать бота в Telegram (5 минут, по кнопкам)

1
Открой Telegram, в поиске введи @BotFather, открой его и нажми Start.
2
Отправь команду /newbot.
3
Введи имя бота (любое, напр. StaffSwipe) → потом username, обязательно заканчивается на bot (напр. staffswipe_bot).
4
BotFather пришлёт строку вида 123456:AA.... — это TELEGRAM_BOT_TOKEN. Скопируй и сохрани (понадобится в Части 4).
Привязку Mini App к боту сделаем в Части 6, когда будет адрес сайта.

💸 Часть 2. Деньги и налоги (как принимать оплату легально)

Чтобы принимать платежи через ЮKassa, нужен статус. Самый простой старт — самозанятый.

Шаг А. Стать самозанятым (10 минут)

  1. Установи приложение «Мой налог» (ФНС) или зайди через Госуслуги.
  2. Зарегистрируйся как плательщик НПД (самозанятость). Это бесплатно.

Шаг Б. Подключить ЮKassa

  1. Зарегистрируйся на yookassa.ru, выбери тип «Самозанятый».
  2. После модерации получишь shopId и секретный ключ — это YOOKASSA_SHOP_ID и YOOKASSA_SECRET_KEY (в Часть 4).
  3. В настройках ЮKassa укажи адрес уведомлений (вебхук) — см. Часть 6.
Про налоги — два разных потока, не путай:
  • Налог исполнителей (тех, кто берёт смены): они самозанятые и платят свои 4%/6% сами через «Мой налог». Приложение лишь печатает им акт — это не твой налог.
  • Твой налог (с подписок/Stars/комиссий): как самозанятый платишь 6% с поступлений от юрлиц/ИП (заведения) и 4% — от физлиц. Лимит дохода НПД — 2,4 млн ₽/год.
Перерастёшь лимит или захочешь нанять людей — оформляй ИП на УСН «Доходы» 6%. Перед стартом сверься с бухгалтером — это ориентир, не консультация.

🖥️ Часть 3. Поднять сервер (технический шаг — можно отдать программисту)

Нужен любой VPS в России (Timeweb, Selectel, VK Cloud, Яндекс.Облако) от ~500 ₽/мес, с установленным Docker. Дальше — три команды. Передай их программисту, если сам не хочешь.

git clone <адрес-репозитория> staffswipe
cd staffswipe/backend
cp .env.example .env        # затем заполни .env (Часть 4)
docker compose up -d        # поднимет базу, API и бота
  • API поднимется на порту 8000, проверка — /health должен отвечать {"status":"ok"}.
  • Нужен домен и HTTPS (бесплатный сертификат Let's Encrypt) — обязательное условие Telegram Mini App.

Фронтенд (Mini App)

cd ../tma
cp .env.example .env        # пропиши VITE_API_BASE_URL и VITE_USE_BACKEND=true
npm install && npm run build # готовая папка dist/ — выложи на хостинг/CDN по HTTPS
Адрес, где открывается dist/ по HTTPS — это и есть MINI_APP_URL. Его же дашь BotFather в Части 6.

🔑 Часть 4. Куда какие ключи вставить (по буквам)

Открой файл backend/.env и впиши значения после знака = (без пробелов, без кавычек). Вот что обязательно для боевого запуска:

Строка в .envЧто вписать
DEV_MODE=falseобяз.
JWT_SECRET=длинная случайная строка (30+ символов)обяз.
INTERNAL_API_SECRET=ещё одна случайная строкаобяз.
ALLOW_INSECURE_TELEGRAM_AUTH=falseобяз.
TELEGRAM_BOT_TOKEN=токен из Части 1обяз.
BOT_USERNAME=username бота без @ (напр. staffswipe_bot)обяз.
MINI_APP_URL=HTTPS-адрес Mini App (Часть 3)обяз.
ALLOWED_ORIGINS=тот же HTTPS-адрес Mini Appобяз.
YOOKASSA_SHOP_ID=из кабинета ЮKassaдля рублей
YOOKASSA_SECRET_KEY=из кабинета ЮKassaдля рублей
PAYMENT_RETURN_URL=адрес Mini App (куда вернуть после оплаты)для рублей
DADATA_TOKEN=токен DaData (бесплатный тариф)опц.
S3_ENDPOINT / S3_BUCKET / S3_KEY / S3_SECRET / S3_PUBLIC_BASEданные Yandex Object Storage (фото)опц.
ADMIN_TG_IDS=твой Telegram-id (узнать у @userinfobot) — доступ к аналитикеопц.
Защита от ошибки: если поставишь DEV_MODE=false, но забудешь сменить JWT_SECRET или INTERNAL_API_SECRET — сервер специально не запустится и напишет, что не так. Так нельзя случайно выложить дырявую версию.
Как сгенерировать случайную строку: на сервере выполни openssl rand -hex 24 — скопируй результат.

▶️ Часть 5. Запустить

cd staffswipe/backend
docker compose up -d         # старт
docker compose logs -f api   # смотреть логи (Ctrl+C чтобы выйти)

Проверь в браузере: https://твой-домен/health → должно быть {"status":"ok"}. Если да — бэкенд жив.

🔗 Часть 6. Привязать Mini App и вебхук оплаты

Привязать приложение к боту

  1. Открой @BotFather → /mybots → выбери своего бота.
  2. Bot Settings → Menu Button → Configure menu button.
  3. Вставь MINI_APP_URL и задай подпись кнопки (напр. «Открыть StaffSwipe»).

Вебхук ЮKassa (чтобы оплаты зачислялись)

  1. В кабинете ЮKassa → «Уведомления» (HTTP-уведомления).
  2. Укажи адрес: https://твой-домен/billing/yookassa/webhook?secret=ЗНАЧЕНИЕ_INTERNAL_API_SECRET
  3. Включи событие payment.succeeded.
Telegram Stars подключать отдельно не нужно — работает через токен бота.

✅ Часть 7. Проверить, что всё работает

  1. Открой бота в Telegram → нажми кнопку меню → приложение открылось.
  2. Зайди как заведение → создай тестовую вакансию (укажи город) → «Опубликовать».
  3. Зайди как соискатель (с другого аккаунта) → выбери тот же город → вакансия в ленте.
  4. Свайп вправо → лайкни в ответ заведением → появился мэтч и чат.
  5. Подтверди смену с обеих сторон → сформируй PDF-акт.
  6. (Если подключил ЮKassa) проведи тестовую оплату подписки.
Прошло — продукт боевой. Можно вести людей.

🏙️ Часть 8. Наполнить ленту и мультигород (самое важное!)

Без реальных вакансий лента пустая и люди уходят. Поэтому:

  1. Выбери ОДИН город / район для старта.
  2. Обойди/обзвони 30–50 заведений, заведи их смены сам через роль «заведение» (укажи город каждой смены).
  3. Только когда лента полная — запускай рекламу на соискателей.
Про твой вопрос «зайдут из другого города»: это уже учтено. Когда рилсы увидит человек из другого города — он выберет свой город и увидит свою ленту. Но если в его городе вакансий нет, он увидит «в городе пока нет смен». Поэтому сначала концентрируйся на одном городе, а в рекламе пиши, какой это город. Расширяться на новые города — по одному, повторяя пункт 2.

📣 Часть 9. Как продвигать (у тебя уже есть рилсы и люди)

  • Рилсы/TikTok: «нашёл смену на сегодня за 1 свайп» — но с привязкой к городу: «Работа в общепите [город]». Так не сольёшь охват на тех, кому нечего показать.
  • Сотрудники/команда: пусть первые недели вручную заводят вакансии и зовут заведения — это тот самый «ручной старт», который запускает маркетплейс.
  • Telegram-чаты «Работа [город]», студенческие и общежитские чаты — там целевые соискатели.
  • Реферальная механика уже в коде — соискатель зовёт соискателя за супер-лайки.
  • Заведениям — первый месяц Pro бесплатно: «бесплатно закрою срочную смену за час».
  • Плотность важнее охвата: добей один город до состояния «лента всегда полная», потом следующий.

💰 Часть 10. Сколько можно заработать (честно)

Заработок = число платящих заведений. Средний платящий ≈ 2 000–2 500 ₽/мес (Pro/Business + boost). Это не гарантия, а зависимость:

СценарийПлатящих заведенийВыручка/мес (до налога и комиссий)
Без наполнения ленты0–30–6 тыс ₽
Пилот удался (1 район, 3–6 мес труда)15–2530–60 тыс ₽
Растущий (город, 1 продажник)80–120200–300 тыс ₽
Зрелый (город+, команда, год-два)300+600 тыс+ ₽

Чистыми из «пилот удался» (~40 тыс) после комиссии ЮKassa (~3,5%), сервера (~2 тыс) и налога 6% остаётся ~33–35 тыс ₽/мес. Главная ценность пилота — доказать, что модель работает, и дальше масштабировать.

На старте — близко к нулю. Реальные деньги приходят только после доказанного пилота и упорной ручной работы. Кто обещает «миллионы из коробки» — врёт.

❓ Коротко: нужно ли ещё дорабатывать код?

Ответ Нет — для запуска кода достаточно. Дальше точечно и только если пилот пойдёт: карта смен, пуш-напоминания о смене, рейтинг исполнителей. Сейчас главное — выйти к первым заведениям, а не добавлять фичи.
