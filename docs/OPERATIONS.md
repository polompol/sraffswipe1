# Операционная шпаргалка: все вопросы, которые возникнут в работе

Живой FAQ оператора StaffSwipe. Дополняй по мере появления новых случаев.

## Деньги: где физически лежат

| Что | Где лежит |
| --- | --- |
| Оплата смены работнику | Идёт **напрямую** заведение → работник (наличные/перевод в день смены). Через сервис не проходит вообще |
| Комиссия сервиса | Реальные деньги приходят **на твою карту/счёт** (СБП как самозанятому; с ЮKassa — на счёт магазина ЮKassa → выплаты на твой расчётный счёт) |
| «Баланс» заведения в приложении | Это **учётная запись в твоей БД** (сколько аванса заведение тебе уже перевело и сколько списано). Сами деньги — у тебя; БД хранит только цифры и журнал движений (`wallet_txns`) |

Отсюда правило: приложение никогда не «теряет деньги» — оно может максимум
разойтись с реальностью в учёте, и тогда сверяешься по журналу движений и
выписке банка.

## Персональные данные (регистрация, ИНН, 152-ФЗ)

- **Что храним**: имя, дата рождения (18+), город/район, роли, статус
  медкнижки (только да/нет/просрочена — БЕЗ фото документов), ИНН
  (самозанятого — для акта; заведения — для верификации), фото профиля,
  tg_id. Телефон — только при входе по SMS.
- **Где**: PostgreSQL на ТВОЁМ сервере. Хостинг обязательно российский
  (Timeweb/Selectel — Москва) — этим выполняется требование 152-ФЗ о
  локализации ПДн в РФ.
- **Кто видит**: ИНН не отдаётся никому, кроме владельца и админа (лента
  кандидатов отдаёт профиль без ИНН/телефона). Админ-панель — только
  tg_id из `ADMIN_TG_IDS`.
- **Что сделать один раз**: подать уведомление оператора ПДн в РКН
  (бесплатно, онлайн-форма на pd.rkn.gov.ru) — формальность, но при жалобе
  проверяют первым делом. Политика и оферта уже в `docs/legal/`.
- Пользователь просит удалить данные: заблокируй аккаунт в админке и удали
  строку из БД (`DELETE FROM users WHERE id=...`) — по 152-ФЗ на это 30 дней.

## Идентичность клиента: «как понять, что это тот же человек»

Якорь личности — **tg_id** (числовой id Telegram-аккаунта, не меняется
никогда, в отличие от @ника и номера). Один Telegram = один аккаунт роли.

| Случай | Что делать |
| --- | --- |
| Потерял доступ к Telegram / завёл новый | Человек пишет в поддержку с нового аккаунта → просишь назвать данные старого профиля (имя, ИНН, последние смены — то, что знает только он) → админка → найти старый аккаунт → «↔ Новый Telegram» → ввести новый tg_id. Рейтинг, история смен и баланс переезжают; пустой дубль блокируется автоматически |
| Сменил номер телефона | Ничего: вход по Telegram, номер не участвует |
| Заведение сменило владельца/управляющего | Тот же перенос на tg_id нового управляющего; ИНН заведения остаётся |
| Забаненный пытается вернуться с нового Telegram | Новый аккаунт формально чистый. Ловится по совпадению ИНН/названия/адреса и по жалобам. Увидел — бан. Идеальной защиты нет ни у кого (даже у Яндекса) |
| Один человек = работник И владелец кафе | Нормально: роли живут в разных таблицах, может иметь оба профиля |

## Частые операционные ситуации

1. **«Не приходят уведомления»** — человек заблокировал бота. Пусть напишет
   боту /start заново.
2. **«Не могу отметиться на смене»** — кнопка «Проблема» уже в приложении →
   спор → ты решаешь двумя кнопками. Правило: код введён верно = работник
   был на месте.
3. **«Оплатил комиссию, а блок висит»** — проверь, нажал ли ты «Оплачено» в
   админке. Блок снимается сам в ту же секунду.
4. **«Заведение просит отсрочку»** — можно: «Оплачено» без денег (разовое
   прощение фиксируй для себя), или пусть пополнит баланс частично.
5. **Переплата/двойной перевод** — зачисли лишнее на баланс («Баланс +N ₽») —
   спишется само со следующих смен. Это проще возврата.
6. **Несовершеннолетний зарегистрировался, наврав про возраст** — по жалобе
   бан. Дата рождения проверяется на сервере (18+), но паспорт мы не видим —
   ответственность в оферте на пользователе.
7. **Фейковая вакансия / мошенник («пришли предоплату за форму»)** —
   авто-модерация уже флажит подозрительные тексты; жалобы падают тебе.
   Бан заведения + предупреждение работникам в чате смены.
8. **Сервер упал / приложение не открывается** — `docker compose ps`, потом
   `logs -f api`. 95% случаев: кончилось место на диске или не поднялась БД.
   Бэкапы: `docs/DEPLOY.md`, раздел про pg_dump — настрой cron с первого дня.
9. **Наплыв людей из «не того» района** — это не проблема, а карта
   расширения: смотри админку → Источники и город/район в профилях,
   куда идти со сменами дальше.
10. **Заведение хочет счёт на юрлицо с закрывающими** — до ИП работаешь как
    самозанятый (чек из «Мой налог» = закрывающий документ). Юрлицам этого
    обычно достаточно; кому нет — «подключим после оформления ИП».

## Бот: что умеет без Mini App

- `/start` — кнопка запуска приложения (эффект «печатает» включён по
  умолчанию, отключается `BOT_TYPEWRITER=0`).
- `/help` — как работает сервис, шаги.
- `/support` — куда писать при проблеме и как восстановить аккаунт.
- Автоматические личные сообщения: мэтч, подтверждение смены, споры,
  «оплата получена», «баланс пополнен», алерты по сохранённым поискам.
- Оплата Stars (микро-покупки) проходит целиком в боте.

Больше функций в бота НЕ тащим: вся ценность — в приложении, бот это
звонок в дверь. Исключение на будущее: еженедельный дайджест «смены рядом»
рассылкой (инфраструктура уведомлений уже готова).
